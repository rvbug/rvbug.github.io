Why This Project Exists

Setting up a new development machine is tedious. Every developer has experienced the frustration of manually installing tools, configuring editors, and restoring their carefully crafted development environment. Whether you‚Äôre:

  • Getting a new machine
  • Setting up a virtual machine for testing
  • Onboarding a team member
  • Rebuilding after a system crash

You face the same repetitive tasks. This dotfiles project solves that problem with intelligent, cross-platform automation scripts.

What It Does

This project is a complete development environment bootstrapper that:

üéØ Core Features

1. Multi-Platform Support - Automatically detects your operating system (macOS, Ubuntu, Debian, Fedora) - Uses the appropriate package manager (Homebrew, apt-get, dnf/yum) - Handles platform-specific quirks and dependencies

2. Essential Development Tools - Terminal emulators (WezTerm) - Modern shell enhancements (Zsh with Starship prompt) - Advanced editors (Neovim with custom configuration) - Developer utilities (fd, ripgrep, lazygit, tree-sitter) - Version control (Git with optimal settings)

3. Data Science Environment - Optional ML/DS toolkit installation - Sets up isolated Python virtual environments - Installs popular libraries (NumPy, Pandas, scikit-learn, TensorFlow, Jupyter) - Integrates with the cookie-ml project structure

4. Configuration Management - Backs up existing configurations before overwriting - Clones and applies your dotfiles automatically - Manages Neovim configuration from a separate repository - Preserves your customizations with timestamped backups

How It Works

The Installation Flow

1. Entry Point (install.sh)

The main script detects your operating system and calls the appropriate installer:

./install.sh

This simple command kicks off the entire process.

2. Platform-Specific Installation

On macOS (macos.sh): - Installs Homebrew if needed - Reads software_list.txt for tools to install - Handles special cases (Lua 5.4, OCaml) - Offers optional data science toolkit setup

On Linux (linux.sh): - Detects distribution (Ubuntu, Debian, Fedora) - Chooses correct package manager - Safely requests sudo if needed - Adapts package names per distribution - Installs distribution-appropriate alternatives

3. Configuration Deployment

Both platform scripts: 1. Check for existing configuration files 2. Create timestamped backups (e.g., .zshrc.bak.2024-11-20-14:30:00) 3. Clone your dotfiles repository 4. Copy configurations to appropriate locations 5. Clean up temporary files

4. Neovim Setup

Special attention is given to Neovim: - Backs up existing ~/.config/nvim directory - Clones your custom Neovim configuration - Lets Neovim‚Äôs plugin manager handle the rest on first launch

5. Optional Data Science Setup (ds_tools.sh)

If you opt in: - Clones the cookie-ml project structure - Creates an isolated Python virtual environment - Installs scientific computing libraries - Sets up Jupyter Lab/Notebook - Provides a clean project template

Key Design Decisions

Safety First

The scripts never blindly overwrite your data:

# Backup with timestamp before any changes
cp $HOME/.zshrc $HOME/.zshrc.bak.$(date +%Y-%m-%d-%H:%M:%S)

Idempotency

Running the script multiple times is safe - it checks before installing:

if command -v brew &>/dev/null; then
  echo "Homebrew already installed"
else
  echo "Installing homebrew..."
  # Installation command
fi

User Control

The scripts ask before major actions:

read -p "Do you want to install Data Science & ML tools? (y/n): " choice

Intelligent Error Handling

The Linux installer adapts to your privileges:

read -p "do you want to run this script as sudo (y/n)?" choice
case "$choice" in
  y|Y ) su_user=sudo
        pkg_mgr="$su_user $pkg_mgr"
        ;;
  n|N ) su_user=" "
        echo "running as normal user..."
        ;;
esac

Getting Started

Prerequisites

  • Git must be installed (or the script will install it)
  • For Linux: ability to install packages (sudo recommended)
  • For macOS: Xcode Command Line Tools (script installs if needed)

Quick Start

# Clone the repository
git clone https://github.com/rvbug/.dotfiles.git
cd .dotfiles

# Make scripts executable
chmod +x *.sh

# Run the installer
./install.sh

What to Expect

The installation process will:

  1. Detect your system - Takes seconds
  2. Update package lists - If you choose to (1-2 minutes)
  3. Install core tools - 5-15 minutes depending on your connection
  4. Configure dotfiles - Less than a minute
  5. Setup Neovim - 30 seconds plus plugin installation on first launch
  6. Optional: DS toolkit - 10-20 minutes for full Python stack

After Installation

  1. Close and reopen your terminal to load new configurations
  2. Launch Neovim: nvim (wait for plugins to install)
  3. Verify installations: brew list (macOS) or apt list --installed (Debian/Ubuntu)
  4. Activate DS environment: cd ~/ml-cookie-cutter && source venv/bin/activate
